# Portfolio – mode d’emploi

Ce dossier contient un portfolio **prêt à publier** (GitHub Pages, Netlify, Vercel).

## 1) Personnaliser (10–20 min)
- Ouvrir `index.html` et remplacer :
  - **Votre Nom**, email, téléphone, liens GitHub/LinkedIn
  - Les textes des sections (héros, compétences, parcours)
- Mettre votre CV en PDF à la racine et le nommer **`CV.pdf`**.
- Mettre une photo dans `assets/` (optionnel) et changer `avatar.svg` dans `index.html`.

## 2) Études de cas
Trois exemples sont dans `cases/`. Copiez `case-template.html` pour chaque projet réel.
Structure conseillée (remplacez les ⚙️ par vos infos) :
- **Contexte** – pour qui, quel problème métier ?
- **Rôle & contraintes** – ce que vous avez fait, limites (temps, données…)
- **Solution** – architecture, choix techniques, captures d’écran
- **Impact** – chiffres concrets (ex. -30% temps de saisie), ⚙️ tests, ⚙️ métriques
- **Liens** – Démo, repo, ticket d’issues notables

## 3) Déploiement GitHub Pages (facile)
1. Créez un repo *public* `portfolio` et uploadez tous les fichiers.
2. Dans **Settings → Pages**, Source = **Deploy from a branch**, Branche = **main**, Dossier = **/**.
3. Attendez 1–2 minutes : votre site sera en ligne à l’URL de votre repo.

## 4) Conseils impact
- Ajoutez une **courte vidéo** (2 min) par projet : *problème → solution → résultat*.
- Mettez les **métriques** en gras dans chaque étude de cas.
- Relisez à voix haute : phrases courtes, verbe d’action, concret > buzzwords.
